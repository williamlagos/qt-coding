Zhockon Platform Documentation
Zhockon is a game development platform written in Python, CPython and C++. It itends to be an alternative to DirectX+XNA, by the use of technologies OpenGL, OpenAL, SDL and Qt, integrated with technologies such as OGRE, Blender, OSG, and much more.

The platform is basically divided into a simple element called Module. Internal libs and apps are made by them. They are grouped by Packages and can be derived to a Library, interacting with low-level access to SDL+OGL+OAL and other loaded libraries, or to a Binding, accessing interfaces such as Python, Qt and Web, like the example above:

  • Zhockon::Audio::OpenAL [Library]
  • Zhockon::Audio::3DSound [Module]
  • Zhockon::Audio::Python [Binding]

Zhockon is one of the three Scopes present on the platform. The other two are Phenomena and Coronae, that provides the same structure of Packages and Modules. So we have the following relation: Scope->Package->Module, where Module can be himself or a Library or a Binding. Simple, isn't it? In code, the scopes are namespaces defined, packages are classes with references to modules objects.

With the Binding derivation, we have application interfaces (APIs) to Qt, Python and Web to all of Packages in a single module, where we have function calls to modules as bridges between Python/C++, Qt/C++ and Python/Javascript. These function calls are optional for both sides and intends to guarantee more performance in interfaces if they are used. The main packages that compose Zhockon are the following:

  • Zhockon::Audio: Simple and 3D sound manipulation
  • Zhockon::Graphics: 2D and 3D animation rendering
  • Zhockon::Input: Input handling
  • Zhockon::Network: Network management
  • Zhockon::Physics: Elements and collisions handling

The Modules are stored in runtime on a main tree, to ease the access and configuration for all of them. We have the same structure to Python API, Qt API and Web API, when they are used separately. The three interfaces can also be combined, for example, Python+Qt, thanks to PySide technology, used as a bridge to communicate both of them.

  • High-Level: Web API <-> Python API <-> Qt API
  • Platform: Coronae+Zhockon+Phenomena
  • Low-Level: OpenGL/OpenAL/SDL

At this moment, the platform is storing information and settings by the ConfigParser and Pickle, but they will be replaced to JSON, the standard serialization format of Zhockon, and the data probably will be managed with an integration between the scopes and a document-oriented database, such as MongoDB or CouchDB, with their respectives interfaces.
